03 / 11
The Problem

Generic APMs cannot monitor AI agents. At any scale.

Traditional monitoring tools track HTTP requests and server metrics. AI agents think, decide, call tools, spawn sub-agents, read memory, and execute multi-step plans. Generic APMs see network traffic. They cannot see reasoning, decisions, or tool call chains. And when you run agents across multiple frameworks, the fragmentation makes it even worse. You need a fundamentally different approach that works across all of them.
🚫

APMs see requests, not reasoning

HTTP traces do not capture why an agent decided to execute a bash command or which memory it accessed.

🚫

No cross-framework visibility

Each framework logs differently. No tool unifies OpenClaw, NemoClaw, NanoClaw, PicoClaw into one view.

🚫

No sub-agent awareness

Agents spawn sub-agents that spawn sub-agents. Generic APMs have no concept of agent hierarchies across frameworks.

🚫

No token cost attribution at scale

50 agents across 3 frameworks calling different models. No tool tracks per-session, per-model spending across all of them.

Security & Architecture

Zero-knowledge cloud. Plugin architecture. Built to scale.

AES-256-GCM end-to-end encryption. All agent data encrypted on your machine before upload. Our servers store only ciphertext. Even if our database is compromised, your multi-framework agent activity remains unreadable. This is architecture, not a feature toggle.

🔒 Encryption flow

1. Agent data generated locally (any framework)
2. Encrypted with AES-256-GCM on your machine
3. Ciphertext sent to ClawMetry cloud
4. Decrypted only in your browser
5. Key never leaves your device
