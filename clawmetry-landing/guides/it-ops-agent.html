Build an IT Ops Agent

A self-hosted AI agent that monitors your servers, checks uptime, creates incident reports, and responds to alerts. All from Telegram.

What this agent does

  • Monitors server health and uptime across your infrastructure
  • Runs scheduled health checks and alerts you when something is down
  • Creates and tracks incident reports automatically
  • Executes diagnostic commands on your servers
  • Sends you a daily infrastructure summary every morning

Prerequisites

  • A machine to run OpenClaw (any Linux box, Mac, or VPS)
  • An Anthropic API key (get one here)
  • A Telegram bot token (create via BotFather)
  • SSH access to the servers you want to monitor

Step 1: Install OpenClaw

One command to get OpenClaw running on your machine:

curl -fsSL https://openclaw.ai/install.sh | bash

Follow the prompts. Pick your AI model (we recommend Claude Sonnet for IT ops, great balance of speed and cost) and enter your Anthropic API key.

Step 2: Configure Telegram

Open ~/.config/openclaw/config.yaml and add your Telegram bot token:

channels:
  telegram:
    bot_token: "YOUR_BOT_TOKEN_HERE"
    allowed_users:
      - YOUR_TELEGRAM_USER_ID

Start a chat with your bot on Telegram. That's your ops console now.

Step 4: Set up SOUL.md

Create ~/openclaw/SOUL.md to give your agent its mission:

# SOUL.md — IT Operations Agent

You are an IT operations agent. Your job is to keep our
infrastructure running smoothly.

## What you do
- Monitor servers and check uptime on demand
- Run health checks: disk space, memory, CPU, services
- Create incident reports when something goes wrong
- Respond to alerts with diagnostic information
- Send daily infrastructure summaries at 9am

## How you work
- When I say "check servers", run health checks on all
  configured hosts
- Be concise. Status: green/yellow/red for each service
- If something is down, immediately suggest remediation steps
- Log all incidents to memory/incidents/ with timestamps

## Servers to monitor
- prod-web-01: 10.0.1.10 (nginx, app)
- prod-db-01: 10.0.1.20 (postgres)
- prod-cache-01: 10.0.1.30 (redis)

## Model
Use Claude Sonnet for all tasks. Fast, accurate, cost-effective.
