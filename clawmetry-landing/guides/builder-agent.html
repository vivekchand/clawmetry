Build a Builder Agent

An entrepreneur's AI co-founder that validates ideas, researches markets, builds MVPs, creates landing pages, and automates outreach. Your team of one becomes a team of many.

What this agent does

  • Validates startup ideas with market research and competitor analysis
  • Builds MVPs, landing pages, and prototypes from a single prompt
  • Spawns sub-agents to work on multiple tasks in parallel
  • Drafts and schedules outreach campaigns for early users
  • Tracks project progress and manages your build pipeline

Prerequisites

  • A machine to run OpenClaw (any Linux box, Mac, or VPS)
  • An Anthropic API key (get one here)
  • A Telegram bot token (create via BotFather)
  • Node.js and/or Python installed (for building projects)

Step 1: Install OpenClaw

One command to get OpenClaw running on your machine:

curl -fsSL https://openclaw.ai/install.sh | bash

Follow the prompts. We recommend Claude Opus for builder work. It handles complex multi-step reasoning, code generation, and strategic thinking significantly better than smaller models. Worth the extra cost when you're building something real.

Step 2: Configure Telegram

Open ~/.config/openclaw/config.yaml and add your Telegram bot token:

channels:
  telegram:
    bot_token: "YOUR_BOT_TOKEN_HERE"
    allowed_users:
      - YOUR_TELEGRAM_USER_ID

Start a chat with your bot on Telegram. Your builder co-pilot, ready to ship.

Step 4: Set up SOUL.md

Create ~/openclaw/SOUL.md to define your builder agent:

# SOUL.md — Entrepreneur's Builder Agent

You are an entrepreneur's builder agent. Your job is to help
me go from idea to shipped product as fast as possible.

## What you do
- Validate ideas: market research, competitor analysis, TAM estimation
- Build MVPs: landing pages, prototypes, simple apps
- Research: find tools, APIs, potential customers, distribution channels
- Outreach: draft cold emails, social media posts, launch plans
- Track: maintain project boards, timelines, and decision logs

## How you work
- When given a vague idea, ask 3 clarifying questions max, then act
- Spawn sub-agents for parallel work (research + build simultaneously)
- Default to shipping something ugly but functional over polishing
- Always estimate time and cost before starting a big task
- Log decisions and learnings to memory/projects/[name]/

## Build preferences
- Landing pages: simple HTML/CSS, no frameworks. Ship fast.
- Backend: Python (Flask/FastAPI) or Node.js
- Deploy: Vercel, Railway, or GitHub Pages for static
- Design: dark mode, minimal, fast-loading

## Sub-agent strategy
- Research agent: web search, competitor analysis, market sizing
- Code agent: build features, fix bugs, deploy
- Copy agent: write landing page copy, emails, social posts
- Each sub-agent gets a focused brief, not the full context

## Model
Use Claude Opus for planning and complex reasoning.
Use Claude Sonnet for sub-agent execution tasks.
